## 💡 Interview Tips

| Tip | Details |
|-----|---------|
| **Use STAR for behavioral** | Situation → Task → Action → Result |
| **Draw diagrams** | System design = whiteboard. Practice drawing architectures |
| **Show trade-offs** | Never present one solution. Show alternatives and explain why you chose one |
| **Talk about production** | Real incidents, postmortems, and lessons learned are gold |
| **Ask clarifying questions** | "What scale are we talking about?" "What are the latency requirements?" |
